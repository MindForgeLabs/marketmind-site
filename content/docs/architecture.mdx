# Architecture

MarketMind is currently a governed research substrate, not an execution-ready trading platform.

## Current Platform

The current architecture centers on:

- Point-in-time-safe data handling.
- Leakage-aware validation boundaries.
- Deterministic run artifacts.
- Canonical bundle and provenance discipline.
- Statistical and governance gates.

These pieces exist to make research claims reviewable before any candidate is promoted.

## Proposed Future Lanes

The following are roadmap lanes, not current delivered product truth:

- Adaptive allocator and regime-aware task learning.
- C++/GPU inference runtime.
- Live data feeds, broker adapters, and execution router.
- Runtime risk overlays and live telemetry.

## Ownership Boundaries

The public site owns the narrative and app routing in `apps/web`. Reusable UI primitives live in `packages/ui`. Published docs live in `content/docs`; internal governance notes live in `docs`.

## Promotion Discipline

A candidate capability should only be described as current when its evidence is deterministic, point-in-time safe, provenance-backed, and accepted by the relevant gates. The adaptive allocator has not crossed that bar.
