# MarketMind Documentation

MarketMind is a **validation-first algorithmic trading R&D platform**: governed backtesting, auditable run bundles, gate validation, and reproducible research infrastructure.

These docs are written for:

- Quant researchers and PMs who want regime-aware strategies.
- Infra and platform engineers who care about latency, observability, and safety.
- Power users integrating MarketMind into existing research and execution stacks.

---

## Documentation

The docs follow this structure (same as the sidebar):

1. [Quickstart](/docs/quickstart) – install the client, run health checks, execute a minimal pipeline.
2. [Architecture](/docs/architecture) – high-level system design and core components.
3. [API](/docs/api) – gRPC, REST, and WebSocket interfaces (sub-pages under API).
4. [Strategies](/docs/strategies) – strategy examples and meta-strategy setups.
5. [Risk](/docs/risk) – kill switches, limits, and overlays.
6. [Installation](/docs/installation) – installation and environment setup.
7. [Telemetry](/docs/telemetry) – observability and metrics.
8. [Caching](/docs/caching) – caching behavior and configuration.
9. [ML Pipeline (planned)](/docs/ml-pipeline) – target ML pipeline; not yet shipped.
10. [C++ Runtime (planned)](/docs/cpp-runtime) – target C++ runtime; not yet shipped.

## Where to start

If you want to **see something work in 5–10 minutes**, start with the [Quickstart](/docs/quickstart).

If you want a **system-level picture first**, read [Architecture](/docs/architecture), then use the list above for specific topics.
