# MarketMind Documentation

MarketMind is a **governed research substrate** for systematic trading work. The current public truth is point-in-time-safe data handling, leakage-aware validation, deterministic artifacts, canonical bundle and provenance discipline, and statistical and governance gates.

These docs are written for:

- Researchers who need reproducible backtests and reviewable evidence.
- Engineers who need deterministic artifact and provenance boundaries.
- Reviewers who need to separate current capabilities from proposed allocator and execution work.

## Claim Frame

Every public page should fit one of four categories:

- **What exists today:** governed backtesting, PIT-safe data handling, leakage-aware validation, deterministic artifacts, provenance, and gates.
- **Why it matters:** research claims need repeatable evidence before promotion.
- **What is proposed:** adaptive allocation, richer APIs, live telemetry, and execution runtime work.
- **What must be proven:** superiority over simpler baselines, allocator validity, execution readiness, and operational safety.

The adaptive allocator is a candidate gated by evidence. It is not presented here as a validated product capability.
